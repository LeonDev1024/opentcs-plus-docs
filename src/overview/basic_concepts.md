# 基本概念
本文主要介绍使用系统时需要了解的基本概念。

## 地图模型

OpenTCS 的地图（Plant Model）由一系列基础元素组成，平台通过这些元素来描述 AGV 可通行的拓扑结构、工位以及业务规则。下表列出了常用元素及其含义：

| 名词 | 描述 |
| --- | --- |
| Point（点） | 车辆在地图上的离散坐标节点，用于表示车辆可以停靠、转向或执行动作的位置。点保存自身坐标（x/y/方位）、允许的进出方向、最大速度等属性，是路径连接的端点。 |
| Path（路径） | 连接两个 Point 的有向边，描述车辆在两个点之间的可行驶路线。路径定义长度、最大行驶速度、单/双向属性，以及可选的行驶曲线，用于约束调度器的路径规划。 |
| Location（位置） | 代表实际业务工位（如充电桩、装卸工位）。Location 会关联到某个 Point 或多个 Point，用于指示车辆在执行订单时需要到达的物理位置。位置本身不存储几何信息，而是通过 `linkedPoints` 关联到点。 |
| LocationType（位置类型） | 描述 Location 的类别及其可执行的操作，例如“装载工位”包含 `Load` 动作，“充电站”包含 `Charge` 动作。通过 LocationType 可以批量定义同类 Location 的动作能力以及自定义属性。 |
| Block（块） | 一组 Point / Path 的集合，用来表示同一段逻辑区域，例如只能同时进入一辆车的狭窄通道。调度器在计算路由时会确保 Block 内资源互斥，避免死锁与冲突。 |
| Layer（图层） | 用于在可视化编辑器中分离不同的地图元素集合，例如“道路层”“传感器层”。图层不影响调度逻辑，但可以控制编辑器中的显示/隐藏与锁定，方便维护复杂地图。 |
| Layer Group（图层组） | 多个 Layer 的组合，用于成批管理显示状态。通过 Layer Group 可以快速切换某类元素（例如全部基础设施层）的可见性，在调试和培训场景下更高效地查看模型。 |

这些元素共同构成了调度系统的拓扑模型，编辑地图时应先规划 Point/Path 的拓扑，再补充业务相关的 Location 及类型，最后根据需要通过 Block、Layer 等机制加强运行约束与可视化管理。
